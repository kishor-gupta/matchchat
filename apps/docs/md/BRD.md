# MatchChat — Business Requirements Document (BRD)

## 1. Project Overview
MatchChat is a 1-to-1 random chat web application that allows users to instantly connect with a random person without submitting any personal information.

## 2. Objectives
- Provide an anonymous, instant, and frictionless chatting experience.
- Connect users with one click through a simple “Start Chat” flow.
- Ensure the platform is fast, scalable, and safe.

## 3. Key Features

### 3.1 Core Flow
- User clicks **Start Chat**.
- System pairs the user with a random active user.
- User can **Skip** anytime to get matched with another user.
- No login or signup required.

## 4. Data & Privacy
No personal user information will be collected.

Only non-personal, system-level analytics may be stored:
- Active user count
- Total chats started
- Number of skips
- Browser type (optional)
- Country (optional, based on IP — IP will NOT be stored)

## 5. Non-Functional Requirements
- High availability
- Real-time messaging
- Zero personal data storage
- Scalable backend to support high traffic

## 6. Platforms
- Web only (mobile responsive)

## 7. Success Metrics
- Daily active users (DAU)
- Total chats initiated
- Average chat duration
- Skip rate
